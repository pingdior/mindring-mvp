探针卡组（Socratic Probe Deck）详细问题库

用途：在Weekly Deep Dive与Topic Sprint中,Alex基于用户当周内容与特征,选择合适探针卡以引导"发散-对抗-收敛"。每张卡包含：触发条件、示例问题、后续推进、对应可测特征。

反例检验（Counterexample）

  • 触发：用户出现强断言、少有反证；多视角度偏低。
  • 示例问题：
    • "如果你的核心判断是错的,最可能错在哪里？有没有历史案例支持这个反例？"
    • "有哪些条件成立时,你的结论会不再成立？"
    • "站在主要反对者视角,他们的最佳论据是什么？"
  • 后续推进：要求至少一次具体反证＋一个小型验证行动。
  • 对应特征：multi_perspective↑premise_explicitness↑time_linkage↑

迁移类比（Analogy Transfer）

  • 触发：主题陷于本域；缺少跨域连接；创新度不足。
  • 示例问题：
    • "在其他领域有相似问题被如何解决？有哪些可迁移的结构或约束？"
    • "选一个完全不同的行业/艺术/自然系统,映射你的问题会得到什么启发？"
    • "如果把你的问题看成'生态系统',哪些物种/资源/竞争关系对应你当前的要素？"
  • 后续推进：产出2个可执行的跨域启发方案,选1个试验。
  • 对应特征：multi_perspective↑生发度↑

约束重构（Constraint Reframe）

  • 触发：资源/时间受限导致停滞；抱怨约束但未重构策略。
  • 示例问题：
    • "如果预算只有现在的20%,你会如何设计最小可行方案？"
    • "把关键约束当作设计素材：能否利用它形成差异化？"
    • "如果只能在两周内完成,你的路线图是什么？"
  • 后续推进：产出一个"约束驱动"的方案与风险假设清单。
  • 对应特征：premise_explicitness↑整合度↑

角色互换（Role Reversal）

  • 触发：他人视角理解不足；外在觉察弱；关系张力。
  • 示例问题：
    • "如果你是对方（同事/用户/伴侣）,你会如何评价你的方案？最在意的风险是什么？"
    • "对方做出当前行为的'善意解释'可能是什么？"
    • "对方的成功指标是什么？你的方案满足了吗？"
  • 后续推进：列出对方三项动机或约束,调整自身方案。
  • 对应特征：multi_perspective↑自我觉察度↑

降维/本质化（Reduction to Essence）

  • 触发：问题表述复杂、枝叶繁多；前提未显性化。
  • 示例问题：
    • "把问题压缩为一句'命题'：你究竟在试图证明什么？"
    • "列出三个不可缺少的变量,其余都是噪音？"
    • "如果只允许一个行动来测试本质,你会选哪个？"
  • 后续推进：产出"命题-前提-证据"的最小证据链。
  • 对应特征：premise_explicitness↑整合度↑

边界与能量（Boundary & Energy）

  • 触发：情绪耗竭、混淆自他情绪、高敏感者过载。
  • 示例问题：
    • "这情绪是你的还是他人的？证据是什么？"
    • "本周你的能量来源与耗损各是什么？如何做一个'能量预算'？"
    • "哪一个'拒绝'可以为你带来最大净收益？"
  • 后续推进：制定一项边界练习与一次情绪清空仪式。
  • 对应特征：自我觉察度↑

风险前视（Forward Risking / Ni训练）

  • 触发：短视冲动（ESTP/ESFP倾向）,缺少长期后果评估。
  • 示例问题：
    • "这个决定在1天/1月/1年后的可能影响是什么？"
    • "最坏情况是什么？你的应对预案？"
    • "有哪些早期信号可以告诉你需要调整？"
  • 后续推进：形成'三时段影响表'与'最坏情况对策'。
  • 对应特征：整合度↑time_linkage↑

真实与脆弱（Authenticity & Vulnerability）

  • 触发：过度"解决方案化",忽视情感（ENTJ/三号成就者）。
  • 示例问题：
    • "你此刻真正的感受是什么？若不解决,只是陪伴,你会如何表达？"
    • "你把价值与成就绑定的证据来自哪里？若不表现,你仍有价值吗？"
    • "写一段'非解决方案'的回应给重要的人。"
  • 后续推进：一次"无方案的陪伴"练习与反思。
  • 对应特征：自我觉察度↑

O-Index 特征的正反示例集

目的：为特征抽取提供"正例/反例/边界例"的文本示范,便于规则迭代与标注训练；为用户透明解释提供可视化证据。

特征A：多视角度（Multi-Perspective）

定义：同一主题中出现支持、反驳、替代方案或界限条件的表达。

✓ 正例：

  • "我支持X,因为A/B。但另一方面,Y可能导致失败。替代方案是Z。"
  • "在资源充足时X成立；若预算减半,该结论不一定成立（界限条件）。"

✗ 反例：

  • "X就是对的。"（无任何反证或替代）

⚠ 边界例：

  • "有人不同意,但我不在乎。"（提到反对但未给论据,不计分或低权重）

特征B：前提显性化率（Premise Explicitness）

定义：显式说明假设/前提/条件的比例。

✓ 正例：

  • "前提是我们有A资源；如果A缺失,则策略需改为B。"
  • "在假设C成立的情况下,我的结论为D。"

✗ 反例：

  • "我们就这么做吧。"（无前提或条件说明）

⚠ 边界例：

  • "应该可以。"（暗含假设但未明确,低权重）

特征C：跨时间引用（Time Linkage）

定义：引用历史记录或跨时间对比的次数与跨度。

✓ 正例：

  • "这让我想起半年前的记录 #rec045（链接）,当时我认为……"
  • "与上周的实验相比,本周结果更稳。"

✗ 反例：

  • 完全不引用或对比任何过去记录。

⚠ 边界例：

  • "上次好像也这样。"（未绑定具体记录,低权重）

特征D：情绪精细度（Emotion Granularity）

定义：从粗糙到细粒度的情绪命名与区分能力。

✓ 正例：

  • "我不是'不开心',而是'失望＋轻微焦虑',同时对未知有'好奇'。"
  • "愤怒里夹杂着不被尊重的感觉（羞辱）,强度7/10,持续约2小时。"

✗ 反例：

  • "心情不好。"（模糊）

⚠ 边界例：

  • 使用一个情绪词但与行为不匹配（需要纠偏机制）。

特征E：整合度（Integration Depth）

定义：是否形成"命题-前提-证据-反证-行动"的结构化链条。

✓ 正例：

  • "命题X；前提A/B；证据C；反证D；行动E（两周内检验）。"

✗ 反例：

  • "我有个想法X。"（无结构）

⚠ 边界例：

  • "命题X；证据C。"（缺前提/反证/行动,部分积分）

特征F：动机透明度（Motivation Transparency）

定义：对行动背后真实动机的自我揭示（承认认可驱动/回避恐惧等）。

✓ 正例：

  • "我急于给方案不是为了效率,而是害怕被看作无用。"
  • "我拒绝这个合作,主要动机是保持自主性,而非商业利益。"

✗ 反例：

  • 给出表层理由,不承认更深动机（常见防御）。

⚠ 边界例：

  • "也许是因为面子吧。"（不明确,低权重）

📐 "如何计算"说明（示例）

多视角度如何计算？

  • 我们在你的文本中识别"支持/反驳/替代/界限条件"的表达（例如"但是/另一方面/在…条件下"）。
  • 仅当该表达伴随具体论据或行动时计为有效,否则权重降低。
  • 你可以点击"来源片段"查看具体句子,并标记"误判/不适用"。

前提显性化率如何计算？

  • 识别"前提是/我假设/如果…那么/在…条件下"等句式。
  • 若仅隐含而未明确说明,则不计或低权重。
